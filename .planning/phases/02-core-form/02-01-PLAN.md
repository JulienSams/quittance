# Plan 02-01: Data Model & Types

## Objective

Define TypeScript types and interfaces for the receipt data model (landlord, tenant, property, rental amounts) with French legal field requirements.

## Context

From CONTEXT.md:
- Form will have 4 sections: Propriétaire, Locataire, Bien, Loyer
- Required fields must comply with French legal requirements for "quittance de loyer location meublée"
- Data structure must support PDF generation in Phase 4

<task id="create-types-file">
Create `src/types/receipt.ts` with TypeScript interfaces:

```typescript
// Core data types for receipt generation

export interface Proprietaire {
  nom: string
  prenom: string
  adresse: string
  codePostal: string
  ville: string
  // Optional fields based on research
  telephone?: string
  email?: string
}

export interface Locataire {
  nom: string
  prenom: string
  // Address may differ from property if needed
  telephone?: string
  email?: string
}

export interface Bien {
  adresse: string
  codePostal: string
  ville: string
  complement?: string // apt number, floor, etc.
  typeBien?: string // studio, T2, etc.
}

export interface Loyer {
  loyerHorsCharges: number // ALUR law requires separation
  charges: number
}

export interface ReceiptData {
  proprietaire: Proprietaire
  locataire: Locataire
  bien: Bien
  loyer: Loyer
  // Will be used in Phase 3 for date range
  dateDebut?: Date
  dateFin?: Date
}
```

Adjust fields based on research findings to ensure legal compliance.
</task>
