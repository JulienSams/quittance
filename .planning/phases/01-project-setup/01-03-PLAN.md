# Plan 01-03: Integrate @react-pdf/renderer and Test PDF Generation

## Objective

Install and configure @react-pdf/renderer, set up custom fonts, and create a test PDF that validates French text rendering and formatting capabilities.

## Context

From CONTEXT.md:
- Use @react-pdf/renderer (template-based, JSX to PDF)
- Set up custom fonts for professional output
- Test must validate French text with accents (é, è, à, ô, etc.)
- Test should include basic layout/formatting, not just "Hello World"

From PROJECT.md:
- Frontend-only application (PDF generation must work client-side)
- French legal documents require proper character support
- Professional, print-ready PDF output

## Tasks

<task id="1">
Install @react-pdf/renderer:
- Run `npm install @react-pdf/renderer`
- Verify installation completes successfully
</task>

<task id="2">
Select and download custom font files:
- Choose a professional font (e.g., Inter, Roboto, or Open Sans)
- Download regular and bold weights
- Place font files in public/fonts/ directory
- Ensure fonts support French characters (Latin Extended)
</task>

<task id="3">
Create font registration utility:
- Create src/lib/pdf-fonts.ts
- Use Font.register() from @react-pdf/renderer to register custom fonts
- Register both regular and bold weights
- Export font family name for use in styles
</task>

<task id="4">
Create test PDF component:
- Create src/lib/pdf-generator.ts with React PDF Document structure
- Use Document, Page, View, Text components from @react-pdf/renderer
- Apply registered custom fonts to text elements
</task>

<task id="5">
Design test PDF content that validates requirements:
- Include French text with accents: "Quittance de loyer - Propriétaire : Julien Samson - Locataire : François Dupré"
- Test multiple text styles (regular, bold, different sizes)
- Include basic layout elements (sections, spacing)
- Add a simple table-like structure with borders
- Use professional formatting (proper spacing, alignment)
</task>

<task id="6">
Create test UI component:
- Create src/components/TestPDF.tsx
- Add a Button that generates and downloads the test PDF
- Use pdf() from @react-pdf/renderer to generate blob
- Trigger download with proper filename
</task>

<task id="7">
Add test PDF component to App.tsx:
- Import and render TestPDF component
- Add clear label explaining this is a Phase 1 test
- Include instructions for the user
</task>

<task id="8">
Verify PDF generation works:
- Click button to generate PDF
- Open downloaded PDF and verify:
  - French accents render correctly
  - Custom font is applied
  - Text styling (bold, sizes) works
  - Layout structure is correct
  - Professional appearance
  - PDF is printable
</task>

## Verification Criteria

- [ ] @react-pdf/renderer installs without errors
- [ ] Custom fonts load and register successfully
- [ ] Test PDF generates and downloads
- [ ] French accented characters (é, è, à, ô, ç, etc.) render correctly
- [ ] Multiple text styles (bold, sizes) display properly
- [ ] Basic layout/formatting works (spacing, alignment, borders)
- [ ] PDF opens in standard viewers (browser, Preview, Acrobat)
- [ ] PDF looks professional and is print-ready
- [ ] No console errors during PDF generation

## Must Haves

From Phase 1 goal: "Can generate a simple test PDF in browser"

- @react-pdf/renderer is installed and configured
- Custom fonts are registered and working
- Test PDF validates French character support
- Test PDF demonstrates formatting capabilities (styles, layout)
- PDF downloads successfully in the browser
- Output quality is professional and print-ready
- Foundation is established for Phase 4 legal receipt templates

## Notes

This test PDF doesn't need to be a full receipt - it's validating that our PDF generation stack works correctly with French text and professional formatting. The actual receipt template will be built in Phase 4 once we have the data model and form from Phases 2-3.
