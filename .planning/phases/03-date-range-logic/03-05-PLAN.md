# Plan 03-05: Integrate Month Generation with Form State

## Objective

Connect month generation and prorata calculation to the form, generating the complete receipt data structure when dates are selected, and making it available for future PDF generation.

## Context

When user selects start and end dates, automatically generate the list of months with calculated amounts. Store this in form state so it's ready for Phase 4 (PDF generation). This completes the Phase 3 logic layer.

## Tasks

<task id="1">
Extend ReceiptData type to include generated receipts:
```typescript
export interface MonthlyReceipt {
  period: MonthPeriod
  loyerHorsCharges: number
  charges: number
  total: number
  receiptNumber?: string // For PDF generation in Phase 4
}

export interface ReceiptData {
  // ... existing fields ...
  dateDebut?: Date
  dateFin?: Date
  monthlyReceipts?: MonthlyReceipt[] // Generated receipts
}
```
</task>

<task id="2">
Add generateReceipts method to useReceiptForm hook:
- Triggered when both dateDebut and dateFin are set
- Calls generateMonthsInRange(dateDebut, dateFin)
- For each MonthPeriod, calls calculateMonthProrata with form's loyer amounts
- Creates MonthlyReceipt array
- Stores in formData.monthlyReceipts
- Can be triggered manually or automatically on date change
</task>

<task id="3">
Add optional UI feedback for generated receipts:
- Show count of receipts that will be generated (e.g., "12 quittances seront générées")
- Display below date pickers or in Période section
- Could show breakdown: "10 mois complets + 2 mois partiels"
- Keep it simple — detailed preview is Phase 5 scope
</task>

<task id="4">
Handle edge cases and validation:
- If dates change after receipts generated, regenerate automatically
- If loyer amounts change, receipts should regenerate if dates exist
- Validation: ensure receipts are generated before allowing save (or handle gracefully)
- Store monthlyReceipts in localStorage with rest of form data
</task>

<task id="5">
Update storage serialization for MonthlyReceipt array:
- MonthlyReceipt contains MonthPeriod which has Date objects
- Ensure Dates in nested structure serialize/deserialize correctly
- Test: save form with dates, reload, verify monthlyReceipts restored correctly
</task>
