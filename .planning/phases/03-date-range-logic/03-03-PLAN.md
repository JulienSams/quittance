# Plan 03-03: Implement Month Generation Logic

## Objective

Create utility functions to generate a list of all months between start and end dates, detecting partial months and preparing month-by-month data structure for receipt generation.

## Context

Given start and end dates, generate an array of month objects representing each month in the range. Each month object contains year, month number, whether it's partial, and the date range it covers. This is the core temporal logic that drives receipt generation.

## Tasks

<task id="1">
Create src/lib/date-utils.ts with month generation utilities:
- Function: `generateMonthsInRange(startDate: Date, endDate: Date): MonthPeriod[]`
- MonthPeriod type: { year: number, month: number, startDate: Date, endDate: Date, isPartial: boolean }
- Iterate from start date's month/year to end date's month/year
- For each month, determine actual start/end dates within that month
</task>

<task id="2">
Implement partial month detection:
- First month is partial if startDate.getDate() !== 1
- Last month is partial if endDate is not the last day of that month
- Use helper: `getLastDayOfMonth(year, month)` to check
- Middle months are always full (start = 1st, end = last day)
</task>

<task id="3">
Add MonthPeriod type to src/types/receipt.ts:
```typescript
export interface MonthPeriod {
  year: number
  month: number // 0-11 (JavaScript month index)
  startDate: Date // First day of occupation in this month
  endDate: Date // Last day of occupation in this month
  isPartial: boolean // True if not occupying full month
}
```
</task>

<task id="4">
Handle edge cases:
- Same month period (startDate and endDate in same month): generate single MonthPeriod with isPartial=true
- Year boundaries: correctly transition from December to January
- Leap years: February has 29 days in leap years, 28 otherwise
- Use date-fns functions if needed for date arithmetic
</task>

<task id="5">
Add unit test or manual verification:
- Test case: 2026-01-15 to 2026-03-20 should generate 3 months:
  - January partial (15-31)
  - February full (1-28)
  - March partial (1-20)
- Test same-month: 2026-03-10 to 2026-03-25 = 1 partial month
- Test year boundary: 2025-12-15 to 2026-02-10
</task>
