# Plan 03-02: Add Date Range UI to Receipt Form

## Objective

Add a new "Période" accordion section to ReceiptForm with two date pickers (start and end dates), integrate with form state, and add validation for date ranges.

## Context

The date range section should appear after the Loyer section in the accordion. Two separate Calendar popovers for "Date de début" and "Date de fin". Dates are stored in the existing ReceiptData type which already has optional `dateDebut` and `dateFin` Date fields.

## Tasks

<task id="1">
Add "Période" section to ReceiptForm.tsx accordion:
- Place after the Loyer AccordionItem
- Title: "Période de génération"
- Add two date picker fields using Calendar + Popover components
- Label first picker: "Date de début"
- Label second picker: "Date de fin"
- Use French locale (fr from date-fns)
- Format displayed dates as "d MMMM yyyy" (e.g., "15 mars 2026")
</task>

<task id="2">
Update useReceiptForm hook to handle date state:
- Add updateDateField method that accepts 'dateDebut' | 'dateFin' and Date | undefined
- Store dates in formData.dateDebut and formData.dateFin
- Dates should auto-save with rest of form when save button clicked
</task>

<task id="3">
Add date validation to validation.ts:
- Create validateDateRange function
- Check: start date is not after end date
- Check: both dates are provided (both or neither)
- Return appropriate French error messages:
  - "La date de début doit être antérieure à la date de fin"
  - "Les deux dates doivent être renseignées"
</task>

<task id="4">
Integrate date validation into form:
- Call validateDateRange in handleBlur for date fields
- Display validation errors inline below date pickers
- Validation should trigger on blur and on save
- Use existing validation error display pattern from Phase 2
</task>

<task id="5">
Update localStorage to handle Date serialization:
- Dates need JSON serialization (toISOString) when saving
- Dates need deserialization (new Date()) when loading
- Update storage.ts saveReceiptData and loadReceiptData to handle Date objects
</task>

## Verification Criteria

- [ ] "Période" accordion section appears after "Loyer"
- [ ] Two date pickers render with French labels
- [ ] Clicking picker shows French calendar (months/days in French)
- [ ] Selected dates display in format "15 mars 2026"
- [ ] Validation error shows if end date < start date
- [ ] Validation error shows if only one date provided
- [ ] Dates persist to localStorage and reload correctly
- [ ] No TypeScript errors with Date types

## Must-Haves

- Two separate date pickers in accordion section
- French locale throughout (labels, calendar, formatting)
- Date validation (start before end, both or neither)
- localStorage persistence working with Date objects
